# 組版アプリ v009

GitHub + Vercel で公開できる、ローカル保存型の簡易組版アプリです。
Next.js・GAS・外部データベースは使用しません。

## v009の主な変更

PDF出力をブラウザの印刷機能から、アプリ内の直接生成方式へ変更しました。

- PCのプリンター設定を使用しません
- ブラウザの印刷余白・倍率・ヘッダー設定に依存しません
- アプリで指定した用紙幅・高さをPDFページサイズへ反映します
- プレビューの改ページ、余白、見出し、空行、字下げをそのまま固定します
- ページ番号、ヘッダー、フッターを反映します
- 余白ガイド、選択中段落の枠、個別設定の編集マークは出力しません
- PDF生成中は、現在のページ数と進捗を画面に表示します

右上の「PDF保存」を押すと、PDFファイルが直接ダウンロードされます。
印刷画面での追加設定は不要です。

## PDFの用途について

v009のPDFは、プレビュー各ページを高解像度画像としてPDFへ固定する方式です。
画面と出力結果の一致を優先しており、確認用PDF、校正用PDF、共有用PDFに適しています。

現段階では次の用途は対象外です。

- PDF内の文字検索・コピー
- PDF上での文章再編集
- PDF/X
- CMYK変換
- 商業印刷所への最終入稿保証
- フォント埋め込み検査

印刷入稿用の文字・ベクターPDFは、実際の印刷会社の仕様確認後に別工程で対応します。

## 既存機能

- 全文編集と章別編集
- `#`、`##`、`###`による見出し設定
- 章の追加・複製・削除・並び替え
- A4、A5、B6、自由用紙サイズ
- 上下左右余白
- フォント、文字サイズ、文字間、行間
- 空行・手動改行の保持
- 本文全体と段落ごとの字下げ制御
- 段落単位の個別調整
- 見出し前の改ページ
- タイトル、ページ番号、ヘッダー、フッター
- 複数プロジェクトのブラウザ保存
- 組版テンプレート
- JSON出力・読込

## v008からの移行

v008と同じVercel URLへ上書きすると、初回起動時に次のデータをv009へコピーします。

- 保存済みプロジェクト
- 現在開いていたプロジェクト
- ユーザー作成テンプレート
- 段落ごとの個別設定
- 章構造
- 組版設定

v008側の保存データは削除されません。

## GitHub・Vercelへの公開

1. ZIPを展開します。
2. 展開したファイルをGitHubリポジトリへ配置します。
3. VercelでGitHubリポジトリを選択します。
4. Framework Presetは `Other` を選択します。
5. Build CommandとOutput Directoryは未設定のままデプロイします。

## 保存時の注意

ブラウザ保存データは、同じ端末・ブラウザ・公開URLでのみ利用できます。
重要な案件は、作業区切りごとにJSONも保存してください。
