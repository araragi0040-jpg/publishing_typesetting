# 組版アプリ v004

出版社向け組版アプリの技術検証版です。Next.js・GAS・外部データベースは使用せず、HTML／CSS／JavaScriptだけで動作します。

## v004の主な変更

本文入力欄の改行情報を、プレビュー・保存データ・PDFへ反映するよう改善しました。

- 1回の改行：同じ段落内の手動改行として保持
- 空行：入力した行数のまま段落間へ反映
- 連続した空行：1行、2行、3行……を個別に保持
- 本文先頭と末尾の空行も保持
- 空行1行分の高さを倍率で調整可能
- 空行保持のオン／オフ切替
- 空行情報をブラウザ保存・JSONへ保存
- v003のプロジェクトとテンプレートをv004へ自動移行
- v003で設定した段落IDと個別調整を可能な範囲で維持

## 現在の主な機能

- タイトル、サブタイトル、著者名、本文の入力
- A4／A5／B6／自由サイズ
- 上下左右余白、フォント、文字サイズ、文字間、行間、字下げ、文字揃え
- 本文の自動改ページ
- 1ページ／見開き表示、倍率変更、余白ガイド
- 複数プロジェクトのブラウザ内保存
- 組版テンプレートの保存・適用・削除
- JSON書き出し・読み込み
- ブラウザ印刷によるPDF出力
- 段落単位の個別調整
  - 文字サイズ
  - 行間
  - 文字間
  - 文字揃え
  - 段落前後の余白
  - 段落前での強制改ページ
  - 次の段落と同じページへの配置
  - 個別設定の解除

## 改行・空行の扱い

### 手動改行

本文内で1回だけ改行した場合、同じ段落内の改行として扱います。

```text
一行目
二行目
```

プレビューでも一行目と二行目の間で改行されます。

### 空行

文章と文章の間に空の行を入れた場合、その行数を保持します。

```text
第一段落

第二段落
```

上記は、第一段落と第二段落の間に空行1行を表示します。

```text
第一段落


第二段落
```

上記は空行2行として表示します。

### 空行の高さ

右側の「本文」設定にある「空行1行の高さ」で調整できます。

- `1.0`：本文の行間と同じ高さ
- `0.8`：通常より少し詰める
- `1.5`：通常より広くする
- `0`：空行情報は保持するが、高さを表示しない

「本文に入力した空行を保持する」をオフにすると、v003と同様に空行を詰めて表示できます。

## 段落の個別調整方法

1. 本文を入力または貼り付けます。
2. 中央プレビュー内の調整したい段落をクリックします。
3. 右側最上部の「段落の個別調整」で設定します。
4. 数値欄を空欄にすると、本文全体の設定を使用します。
5. 「個別設定を解除」で、その段落を本文全体の設定へ戻せます。

選択中の段落には、その段落の前に保持されている空行数も表示されます。

## 保存について

### ブラウザ内保存

プロジェクトとテンプレートは、利用中のブラウザの `localStorage` に保存されます。

- 同じ端末
- 同じブラウザ
- 同じ公開URL

でのみ、そのまま呼び出せます。ブラウザデータを削除すると失われるため、重要な案件はJSONでも保存してください。

### JSON保存

JSONには以下が含まれます。

- 原稿
- 手動改行と空行数
- 全体の組版設定
- 段落ごとの個別設定
- プロジェクト情報

別端末でJSONを読み込むと、新しいプロジェクトとして保存されます。

## PDF出力

1. 右上の「PDF出力」をクリックします。
2. ChromeまたはEdgeの印刷画面で保存先を「PDFに保存」にします。
3. 倍率を100%にします。
4. 余白を「なし」にします。
5. ヘッダーとフッターをオフにします。

ブラウザや端末フォントによって改行位置が変わる場合があります。実際の出版用データと比較しながら検証してください。

## GitHub・Vercelへの公開

1. ZIPを展開します。
2. 中のファイルをGitHubリポジトリの直下へ配置します。
3. Vercelで対象リポジトリをImportします。
4. Framework Presetは `Other` を選択します。
5. Build CommandとOutput Directoryは空欄のままデプロイします。

## 現段階の制限

- 横書き・1段組のみ
- 文字範囲単位の装飾には未対応
- 画像、表、ルビ、傍点、縦書きには未対応
- 印刷入稿用PDF/X、CMYK、トンボには未対応
- 複数人の同時編集には未対応
- 非常に大量の空行を入力した場合、ページをまたぐ空白の分割は簡易処理
- 「次の段落と同じページ」は可能な範囲で段落を次ページへ送る簡易処理

## v003からの移行

同じ公開URLでv003を使用していた場合、初回起動時に以下をv004へコピーします。

- 保存済みプロジェクト
- 現在開いていたプロジェクト
- ユーザー作成テンプレート
- 段落ごとの個別設定

本文の元データから空行数を再解析するため、v003では表示されていなかった空行もv004では反映されます。v003側の保存データは削除しません。
