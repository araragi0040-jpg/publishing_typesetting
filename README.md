# 組版アプリ v008

出版社向けの簡易組版検証アプリです。HTML・CSS・JavaScriptのみで動作し、GitHubからVercelへそのまま公開できます。GASや外部データベースは使用しません。

## v008の方針

機能を増やすことよりも、初めて触る人が迷わず使えることを優先しています。

原稿編集は次の2方式を切り替えられます。

- **全文編集**：Wordなどの既存原稿をまとめて貼り付ける
- **章別編集**：章の一覧から選び、タイトルと本文を個別に編集する

どちらで変更しても同じ原稿データへ自動反映されます。

## v008の追加内容

### 全文編集

- 原稿全文の一括貼り付け
- 大・中・小見出しの挿入ボタン
- `#`、`##`、`###`による見出し指定
- 章別編集との自動同期

### 章別編集

- `# 大見出し`を章タイトルとして自動認識
- 章一覧の表示
- 章ごとのタイトル・本文編集
- 章の追加
- 章の複製
- 章の削除
- 章の前後移動
- 各章の文字数・見出し数表示
- 章内の中見出し・小見出し表示
- 大見出しより前の文章を「冒頭部分」として保持

### 分かりやすさの改善

- 「全文編集」「章別編集」の用途を画面上に表示
- 編集方式は大きな切替ボタンで選択
- 編集内容が常に同期されることを明示
- 見出し記法を画面内に表示
- 章がない場合の操作案内
- 選択中の章と移動可能方向を明確化

## 見出しの記法

```text
# 大見出し・章タイトル
## 中見出し
### 小見出し
```

記号と文字の間には半角スペースを入れてください。

章別編集では、章タイトルが`# 大見出し`に対応します。章本文内で単独の`#`を入力した場合は、誤って別章にならないよう`##`へ変更されます。

## 保存方法

### ブラウザ保存

編集中のプロジェクトはブラウザ内へ自動保存されます。同じ端末・ブラウザ・公開URLで利用してください。

### JSON出力

重要なデータはJSONでも保存してください。JSONには次の内容が含まれます。

- 書籍情報
- 本文全文
- 章構造
- 段落ごとの個別設定
- 組版設定
- プロジェクト情報

### PDF出力

「PDF出力」を押し、ChromeまたはEdgeの印刷画面からPDFとして保存します。

推奨設定：

- 倍率：100%
- 余白：なし
- ブラウザ側のヘッダーとフッター：オフ
- 背景グラフィック：必要に応じてオン

## v007からの移行

同じVercel URLへv008を上書きすると、初回起動時に次のデータをv007からv008へコピーします。

- 保存済みプロジェクト
- 現在のプロジェクト
- 独自テンプレート
- 見出し・空行・段落設定
- ページ番号・ヘッダー・フッター設定

v007側の保存データは削除されません。

## GitHub・Vercelへの公開

1. ZIPを展開します。
2. 展開したファイルをGitHubリポジトリへ配置します。
3. VercelでリポジトリをImportします。
4. Framework Presetは`Other`を選択します。
5. Build CommandとOutput Directoryは空欄でデプロイします。
