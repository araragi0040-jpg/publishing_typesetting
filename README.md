# 組版アプリ v006

出版社向け組版アプリの技術検証版です。Next.js・GAS・外部データベースは使用せず、HTML／CSS／JavaScriptだけで動作します。

## v006の主な変更

本文内の見出しをMarkdown風の記法から自動認識し、見出し階層ごとに組版できるようにしました。

- `# 大見出し`
- `## 中見出し`
- `### 小見出し`
- 大・中・小見出しごとのフォント、文字サイズ、揃え
- 見出し前後の余白
- 見出し前での自動改ページ
- 見出しと直後の本文が別ページになる状態の防止
- タイトル・サブタイトル・著者名の表示／非表示
- タイトルの後から本文を次ページで開始する設定
- 見出し情報をブラウザ保存、JSON、テンプレートへ保存
- v005のプロジェクトとテンプレートをv006へ自動移行

## 見出しの入力方法

本文入力欄で、行頭に見出し記号と半角スペースを入力します。

```text
# 第1章　はじめに

本文が入ります。

## 1. 制作の背景

本文が入ります。

### 補足事項

本文が入ります。
```

見出し記号はプレビューやPDFには表示されません。

以下のように記号の後へスペースを入れていない場合は、通常の本文として扱います。

```text
#見出しとして認識されない文章
```

入力した空行は従来どおり保持されます。見出し前後の空行は、右側で設定した「見出し前後の余白」に追加されます。

## 見出し設定

右側の「本文内見出し」で、大見出し・中見出し・小見出しをそれぞれ設定できます。

- フォント
- サイズ
- 左／中央／右揃え
- 見出し前の余白
- 見出し後の余白
- 見出し前で改ページ
- 直後の本文と同じページに置く

「直後の本文と同じページに置く」をオンにすると、ページ末尾へ見出しだけが残りそうな場合、見出しを次ページへ送ります。

## タイトル部分の設定

右側の「タイトル」で以下を設定できます。

- タイトル、サブタイトル、著者名を誌面へ表示するか
- タイトルの後、本文を次ページから開始するか
- タイトルサイズ
- タイトルと本文の間隔
- タイトルの揃え

入力欄の内容は、誌面表示をオフにしてもプロジェクトデータ内へ保持されます。

## 保存について

### ブラウザ内保存

プロジェクトとテンプレートは、利用中のブラウザの `localStorage` に保存されます。

- 同じ端末
- 同じブラウザ
- 同じ公開URL

でのみ、そのまま呼び出せます。ブラウザデータを削除すると失われるため、重要な案件はJSONでも保存してください。

### JSON保存

JSONには以下が含まれます。

- 原稿
- 見出し階層
- 手動改行と空行数
- 全体の組版設定
- 見出しごとの組版設定
- 段落ごとの個別設定
- プロジェクト情報

別端末でJSONを読み込むと、新しいプロジェクトとして保存されます。

## PDF出力

1. 右上の「PDF出力」をクリックします。
2. ChromeまたはEdgeの印刷画面で保存先を「PDFに保存」にします。
3. 倍率を100%にします。
4. 余白を「なし」にします。
5. ヘッダーとフッターをオフにします。

ブラウザや端末フォントによって改行位置が変わる場合があります。実際の出版用データと比較しながら検証してください。

## GitHub・Vercelへの公開

1. ZIPを展開します。
2. 中のファイルをGitHubリポジトリの直下へ配置します。
3. Vercelで対象リポジトリをImportします。
4. Framework Presetは `Other` を選択します。
5. Build CommandとOutput Directoryは空欄のままデプロイします。

## 現段階の制限

- 横書き・1段組のみ
- 見出し記法は `#` から `###` まで
- 見出しの個別上書きには未対応
- 文字範囲単位の装飾には未対応
- 画像、表、ルビ、傍点、縦書きには未対応
- 印刷入稿用PDF/X、CMYK、トンボには未対応
- 複数人の同時編集には未対応
- 見出しと本文の分離防止は、直後の本文冒頭が収まるかを基準にした簡易処理

## v005からの移行

同じ公開URLでv005を使用していた場合、初回起動時に以下をv006へコピーします。

- 保存済みプロジェクト
- 現在開いていたプロジェクト
- ユーザー作成テンプレート
- 空行・改行情報
- 段落ごとの個別設定
- 段落字下げ設定

v005の本文に `# `、`## `、`### ` で始まる行がある場合、v006では見出しとして再解析されます。v005側の保存データは削除されません。
