# 組版アプリ v007

出版社向け組版アプリの技術検証版です。Next.js、GAS、外部データベースは使用せず、HTML、CSS、JavaScriptだけで動作します。

## v007の追加内容

### 組版設定の開閉表示

右側の設定を、次のカテゴリに分けたアコーディオン形式へ変更しました。

- 段落の個別調整
- 用紙
- 余白
- 本文
- 本文内見出し
- タイトル
- ページ

各カテゴリをクリックして開閉できます。「全て開く」「全て閉じる」も利用できます。開閉状態は同じブラウザ内に保存されます。

本文内見出しは、さらに大見出し、中見出し、小見出しごとに開閉できます。プレビュー上の本文段落をクリックした場合は、「段落の個別調整」が自動的に開きます。

### ページ番号

- 表示／非表示
- 開始番号
- 1ページ目への表示
- 上中央
- 上外側
- 下中央
- 下外側

「外側」は、奇数ページでは右、偶数ページでは左に自動配置されます。

### ヘッダー・フッター

表示内容は次から選択できます。

- タイトル
- 現在の大見出し
- 著者名
- 自由入力

位置は、外側、中央、内側から選択できます。内側・外側は奇数ページと偶数ページで左右が自動的に切り替わります。1ページ目への表示も個別に設定できます。

「現在の大見出し」は、本文内の `# 大見出し` を参照します。

## 見出しの入力方法

本文入力欄で、行頭に見出し記号と半角スペースを入力します。

```text
# 第1章　はじめに

本文が入ります。

## 1. 制作の背景

本文が入ります。

### 補足事項

本文が入ります。
```

見出し記号はプレビューやPDFには表示されません。

## 主な機能

- タイトル、サブタイトル、著者名、本文の入力
- 大見出し、中見出し、小見出しの自動認識
- A4、A5、B6、自由サイズ
- 上下左右余白
- フォント、文字サイズ、文字間、行間
- 段落字下げのオン／オフと字下げ量
- 手動改行と空行の保持
- 本文の自動改ページ
- 見出し前での改ページ
- 見出しと直後の本文の分離防止
- 段落単位の個別調整
- ページ番号、ヘッダー、フッター
- 1ページ／見開き表示
- 複数プロジェクトのブラウザ内保存
- 組版テンプレートの保存、適用、削除
- JSON書き出し、読み込み
- ブラウザ印刷によるPDF出力

## 保存について

プロジェクトとテンプレートは、利用中のブラウザの `localStorage` に保存されます。同じ端末、同じブラウザ、同じ公開URLでのみそのまま呼び出せます。

重要な案件は、作業の区切りごとにJSONでも保存してください。JSONには原稿、組版設定、見出し設定、段落個別設定、ページ番号、ヘッダー、フッター設定が含まれます。

## v006からの移行

同じVercel URLへv007を上書きした場合、初回起動時に次の内容をv006からv007へコピーします。

- 保存済みプロジェクト
- 現在開いていたプロジェクト
- ユーザー作成テンプレート
- 段落個別設定
- 見出し設定
- 改行、空行、字下げ設定

v006側のデータは削除されません。

## GitHub・Vercelへの公開

1. ZIPを展開します。
2. 中のファイルをGitHubリポジトリのルートへ配置します。
3. Vercelで対象リポジトリを読み込みます。
4. Framework Presetは `Other` を選択します。
5. Build CommandとOutput Directoryは空欄のままデプロイします。

既存の同一URLへ反映する場合は、現在のリポジトリへファイルを上書きしてpushします。

## PDF出力時の推奨設定

ChromeまたはEdgeの印刷画面で、次を推奨します。

- 送信先：PDFに保存
- 用紙サイズ：アプリで設定したサイズ
- 余白：なし
- 倍率：100%
- ヘッダーとフッター：オフ
- 背景のグラフィック：必要に応じてオン

ブラウザ標準のヘッダーとフッターをオンにすると、アプリ側のヘッダー・フッターと重なるためオフにしてください。
