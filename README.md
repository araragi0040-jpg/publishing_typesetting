# 組版アプリ v003

出版社向け組版アプリの技術検証版です。Next.js・GAS・外部データベースは使用せず、HTML／CSS／JavaScriptだけで動作します。

## v003の主な機能

- タイトル、サブタイトル、著者名、本文の入力
- A4／A5／B6／自由サイズ
- 上下左右余白、フォント、文字サイズ、文字間、行間、字下げ、文字揃え
- 本文の自動改ページ
- 1ページ／見開き表示、倍率変更、余白ガイド
- 複数プロジェクトのブラウザ内保存
- 組版テンプレートの保存・適用・削除
- JSON書き出し・読み込み
- ブラウザ印刷によるPDF出力
- **段落単位の個別調整**
  - 文字サイズ
  - 行間
  - 文字間
  - 文字揃え
  - 段落前後の余白
  - 段落前での強制改ページ
  - 次の段落と同じページへの配置
  - 個別設定の解除
- v002のプロジェクト・テンプレートを初回起動時に自動移行

## 段落の個別調整方法

1. 本文を入力または貼り付けます。
2. 中央プレビュー内の調整したい段落をクリックします。
3. 右側最上部の「段落の個別調整」で設定します。
4. 数値欄を空欄にすると、本文全体の設定を使用します。
5. 「個別設定を解除」で、その段落を本文全体の設定へ戻せます。

黄色の縦線がある段落は個別設定済み、緑色で強調されている段落は現在選択中です。これらの表示はPDFには出力されません。

## 保存について

### ブラウザ内保存

プロジェクトとテンプレートは、利用中のブラウザの `localStorage` に保存されます。

- 同じ端末
- 同じブラウザ
- 同じ公開URL

でのみ、そのまま呼び出せます。ブラウザデータを削除すると失われるため、重要な案件はJSONでも保存してください。

### JSON保存

JSONには以下が含まれます。

- 原稿
- 全体の組版設定
- 段落ごとの個別設定
- プロジェクト情報

別端末でJSONを読み込むと、新しいプロジェクトとして保存されます。

## PDF出力

1. 右上の「PDF出力」をクリックします。
2. ChromeまたはEdgeの印刷画面で保存先を「PDFに保存」にします。
3. 倍率を100%にします。
4. 余白を「なし」にします。
5. ヘッダーとフッターをオフにします。

ブラウザや端末フォントによって改行位置が変わる場合があります。実際の出版用データと比較しながら検証してください。

## GitHub・Vercelへの公開

1. ZIPを展開します。
2. 中のファイルをGitHubリポジトリの直下へ配置します。
3. Vercelで対象リポジトリをImportします。
4. Framework Presetは `Other` を選択します。
5. Build CommandとOutput Directoryは空欄のままデプロイします。

## 現段階の制限

- 横書き・1段組のみ
- 段落は本文内の空行で判定
- 文字範囲単位の装飾には未対応
- 画像、表、ルビ、傍点、縦書きには未対応
- 印刷入稿用PDF/X、CMYK、トンボには未対応
- 複数人の同時編集には未対応
- 「次の段落と同じページ」は可能な範囲で段落を次ページへ送る簡易処理

## v002からの移行

同じ公開URLでv002を使用していた場合、初回起動時に以下をv003へコピーします。

- 保存済みプロジェクト
- 現在開いていたプロジェクト
- ユーザー作成テンプレート

v002側の保存データは削除しません。
