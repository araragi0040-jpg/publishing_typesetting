# 組版アプリ v016

GitHub / Vercelで公開できる、ブラウザ完結型の簡易組版アプリです。GASや外部データベースは使用しません。

## UI修正

- 書籍構成で複数項目をONにした際、入力欄が縮んで重なる問題を修正しました。
- モーダルの見出しと下部ボタンを固定し、設定部分だけをスクロールする構成に変更しました。
- まえがき・あとがき・著者紹介・奥付をすべてONにしても、各入力欄を順番に操作できます。

## v016の主な追加

- 「書籍構成」画面を追加
- 扉・まえがき・目次・本文・あとがき・著者紹介・奥付の順番を可視化
- まえがきの表示と、目次前／目次後の位置切替
- あとがき・著者紹介の表示
- 奥付の書名、著者、発行日、版・刷、発行者、発行所、連絡先、著作権、備考
- 書籍情報から奥付へタイトル・著者・著作権表記を補完
- 前付・後付が複数ページになる場合の自動改ページ
- 目次内の本文ページ番号を、前付ページ数を含めて再計算
- 横書き・縦書き・直接PDF保存への反映
- v015のプロジェクト・テンプレート自動移行

## 基本操作

1. 上部の「書籍構成」を開きます。
2. 使用する「まえがき」「あとがき」「著者紹介」「奥付」をONにします。
3. 必要な内容だけ入力します。
4. 上部のページ順で配置を確認します。
5. 中央プレビューを確認し、「PDF保存」で出力します。

本文はこれまでどおり「全文編集／章別編集」で管理します。目次は本文中の見出しから自動生成されます。

## ページ順

基本順は次のとおりです。

`扉 → まえがき → 目次 → 本文 → あとがき → 著者紹介 → 奥付`

まえがきのみ、目次の前または後を選択できます。未使用のページは自動的に省略されます。

## 保存について

- 編集中データ：同じブラウザのlocalStorage
- 受け渡し・バックアップ：JSON出力
- 完成確認：アプリ内直接PDF保存

重要な案件は、作業の区切りごとにJSONを保存してください。
