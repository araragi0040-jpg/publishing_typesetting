# 組版アプリ v002

出版社向け組版アプリのローカル運用版です。Next.js、GAS、データベース、ビルド環境を使わず、HTML・CSS・JavaScriptだけで動作します。

## v002の主な追加内容

- 複数プロジェクトをブラウザ内へ個別保存
- 保存済みプロジェクト一覧
- プロジェクトの読込、複製、削除
- 編集中プロジェクトの自動保存
- 組版設定をテンプレートとして保存
- 保存テンプレートの適用、削除
- 標準テンプレート3種類
  - A5 文芸書 基本
  - B6 小説 基本
  - A4 冊子・資料 基本
- v001のブラウザ保存データを初回起動時に自動移行
- JSON読込データを新しいプロジェクトとして保存

## 継続して利用できる機能

- 横書き・一段組
- タイトル、サブタイトル、著者名、本文
- A4、A5、B6、自由サイズ
- 上下左右余白

## 保存データの考え方

v002では、各プロジェクトとテンプレートをブラウザの`localStorage`へ保存します。

保存範囲は次の条件に依存します。

- 同じ端末
- 同じブラウザ
- 同じ公開URL

VercelのURL変更、ブラウザデータの削除、別端末への移動を行うと、そのままでは保存済みプロジェクトを開けません。重要な案件は、作業区切りごとにJSON出力してください。

JSONは次の用途に使用します。

- バックアップ
- 別端末への移動
- 別ブラウザへの移動
- 版ごとの保存
- ブラウザ保存データ消失時の復元

## プロジェクト操作

### 新規

空の原稿と初期組版設定で、新しいプロジェクトを作成します。現在のプロジェクトは先に自動保存されます。

### プロジェクト

保存済みプロジェクト一覧を開きます。

- 開く
- 複製
- 削除

が行えます。

### 保存

現在のプロジェクトを即時保存します。通常は入力後約0.7秒で自動保存されるため、保存ボタンは明示的に保存したい場合に使用します。

### 複製

現在の原稿と組版設定をコピーし、別のプロジェクトとして開きます。

## テンプレート操作

「テンプレート」を開き、名前を入力して「現在の設定を保存」を押します。

テンプレートに保存される内容は次のとおりです。

- 用紙サイズ
- 余白
- 本文書式
- タイトル書式
- ページ番号設定

タイトル、著者名、本文などの原稿内容は保存されません。

テンプレートを適用しても、現在の原稿内容は維持されます。

## ローカルでの確認

`index.html`をChromeまたはEdgeで開いてください。

より本番に近い確認をする場合は、簡易Webサーバーを使用します。

```bash
python -m http.server 8000
```

ブラウザで次を開きます。

```text
http://localhost:8000
```

ファイルを直接開いた場合と、`http://localhost:8000`で開いた場合は保存領域が異なる場合があります。検証中は同じ開き方を継続してください。

## GitHub・Vercelでの公開

1. GitHubで新しいリポジトリを作成します。
2. このフォルダ内のファイルをアップロードします。
3. Vercelで「Add New Project」を選択します。
4. 対象のGitHubリポジトリを選択します。
5. Framework Presetは「Other」を選択します。
6. Build CommandとOutput Directoryは空欄のままデプロイします。

同じVercel本番URLを継続して使用する限り、通常のコード更新ではブラウザ保存データを継続利用できます。

## PDF出力

アプリ右上の「PDF出力」を押します。

Chrome・Edgeの印刷画面では、以下を推奨します。

- 送信先：PDFに保存
- 余白：なし
- 倍率：100%
- 背景のグラフィック：オン
- ヘッダーとフッター：オフ

独自用紙サイズの扱いはブラウザやプリンタードライバーによって異なります。
